## Completed In This Session

### Kin-to-Kin Chat V1

The Kin panel now exposes `Kin間チャット`.

Current behavior:

- v1 supports exactly two saved Kin profiles.
- The user chooses a starter Kin, partner Kin, topic, and max chat count.
- The app sends the starter `SYS_KIN_TO_KIN_CHAT` request to the starter Kin.
- Each Kin reply is parsed as `SYS_<FROM>_TO_<TO>_CHAT` when possible, then
  relayed to the other Kin with app-owned `CHAT_COUNT` metadata.
- The app owns max-count enforcement. Kin-supplied counts are not authoritative.
- Stop waits for the current send/receive cycle to finish, then pauses.
- Reset clears the local transcript and session status.
- At max count, the app sends a final `SYS_INFO` notice to both Kin profiles.
- Optional completion summary sends the transcript to GPT as a summary request.

The max-count input keeps its in-progress edit text separate from the normalized
runtime number, so editing the default `50` down to `2` or `3` works normally.

### Kin Speaker Labels

Kin replies now store the active Kin label in `Message.meta.speakerLabel`.
Message rendering uses that label before falling back to the generic `Kindroid`
display label.

This matters for Kin-to-Kin chat because the ordinary Kin message stream can now
show which Kin produced each reply.

### DB Chunk Pagination

DB document loading now pages through `rag_document_chunks` in 1000-row pages
until exhausted.

The `1000` value is a page size, not a total cap. This avoids the
Supabase/PostgREST default response cap truncating:

- DB tab chunk counts
- exact duplicate detection inputs
- DB organization inputs
- DB compaction inputs
- document-id-specific DB loads

### Library DB Style Warning

`LibraryDbPanel` styles no longer mix `border` shorthand with `borderColor`
overrides in the DB panel style set. Border styles there now use
`borderWidth` / `borderStyle` / `borderColor` so React 19 rerenders should not
emit the shorthand/longhand warning for those elements.

## Current Boundaries

- Kin-to-Kin chat v1 is not a group chat router.
- Do not broaden v1 to more than two Kin profiles without designing routing
  rules and transcript ownership first.
- Do not make Kin-supplied `CHAT_COUNT` authoritative.
- Do not start Kin-to-Kin chat from arbitrary Kin replies; keep it behind the
  explicit Kin panel control.
- Completion summary is a GPT request only. It does not write memory or library
  entries by itself.
