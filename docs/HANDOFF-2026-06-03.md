# Handoff 2026-06-03

## Scope

This closeout covers the Kin-to-Kin chat minimum-turn guard added after the
2026-05-28 compact transfer / reply-translation slice.

## Kin-to-Kin Min Turns

- The Kin-to-Kin drawer now exposes `Min turns` before `Max turns`.
- `Min turns` is clamped to at least `1` and no higher than the resolved
  `Max turns`.
- The active session resolves `minCount` and `maxCount` once at start, so
  in-flight termination behavior does not drift if the inputs are later edited.
- One-to-one Kin chat and facilitator-led group chat both pass
  `MIN_CHAT_COUNT` into the structured Kin instructions.
- Starter / facilitator `**END THE CHAT**` only ends the session after
  `MIN_CHAT_COUNT` has been reached.
- If the starter / facilitator emits `**END THE CHAT**` before `MIN_CHAT_COUNT`,
  the app records the message and continues the chat.
- Participant `**END THE CHAT**` output remains non-terminal.

## Cleanup Notes

- End-token termination now flows through `canEndKinToKinChat(...)`, avoiding
  separate inline starter/minimum-count checks in the one-to-one and group
  runtimes.
- The builder functions are still referenced only by the Kin-to-Kin drawer and
  focused unit tests; no alternate Kin-to-Kin end route was found.
- New user-facing and Kin-facing strings are ASCII protocol/control text, so no
  new mojibake strings were introduced in the touched code.
