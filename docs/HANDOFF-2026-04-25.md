## User-Facing Baseline Now Expected

### Library-card summary toggle

The setting now means:

- `ライブラリカードへの取込時にサマリー自動生成`
- applies across file import, Google Drive import, YouTube transcript import,
  search library saves, and task snapshot saves
- when off, library cards should store no generated summary
- when on, summary-generation usage should be counted in the ingest bucket

### File / Drive import feedback

File import and Google Drive import should now:

- show the shared loading indicator while work is running
- save into the library
- post a visible GPT chat task-info message after import completes
- include extracted character count in that completion message

### PDF / image ingest modes

The user-facing image/PDF choices are:

- `compact`
- `detailed`
- `max`

The internal legacy value for compact image detail is still `simple`. Treat this
as compatibility plumbing, not user-facing copy.

### Ingest output authority

The `/api/ingest` prompt and result selection now intentionally avoid asking
for multiple alternate versions of the same content:

- text-first PDF/image `max`: `rawText` is the authority
- `detailed`: `kinDetailed` is the authority
- `compact`: `kinCompact` is the authority

Generated `compact` and `detailed` lines should not be clipped again by a local
post-selection character budget. The previous clipping path caused final list
items to end with `...` and lose content.
