## Current Direction

The current PPT design flow proved that free-form slide JSON such as
`parts: [{ role, text }]` is too permissive. It encourages vague roles,
overloaded text fields, repetitive layouts, and renderer guesses.

The next implementation should move to a frame-based slide model:

1. Keep upstream information extraction and strategy drafting as rich natural
   language.
2. For each slide, choose one known `frameType`.
3. Fill only that frame's required fields.
4. Treat frame JSON as the canonical slide source.
5. Derive both the user-facing design text and renderer input from that frame
   JSON.

Do not continue by adding more ad hoc prompt rules to the old free-form
`role/text` structure.
