## What Was Finished

### 1. Settings UX split from hanging drawers

- the old single settings button was replaced with 3 icon entry points
- opening those icons now switches the full content area below the title bar into a dedicated settings workspace
- chat settings, task settings, and library settings now live in clearly separated full-area screens
- memory approval UI and SYS rule approval UI were visually unified
- pending approval glow now exists for both:
  - chat icon when memory candidates exist
  - task icon when SYS rule candidates exist

Important: the old hanging drawers for `メモリ`, `トークン`, `タスク`, `ライブラリ` remain in place for compatibility.

### 2. Search and source display

- normal `検索:` requests once again surface source cards in GPT responses
- YouTube source cards now use a muted autoplay preview-style embedded player instead of a static thumbnail

### 3. Task entry unification

- the old `タスク形成` and `タスク進捗` entry points were merged into a single `タスク` drawer entry
- inside that drawer, the UI now switches between task drafting and task progress

### 4. Task draft saving and multiple drafts

- current task draft can now be saved into the library as a neutral information artifact
- snapshots are stored as:
  - summary
  - full text
- multiple task drafts are now retained
- left/right navigation switches the active task draft
- advancing past the last draft creates a new empty slot

### 5. Task progress expansion

- multiple task progress runtimes are now retained
- left/right navigation switches the active runtime view
- `TASK_CONFIRM` with `STATUS: SUSPENDED` is now supported
- progress UI now supports:
  - a visible `ホールド中` state
  - suspend-message generation
  - manual editing of completed count / total count
  - `進捗カウント出力`
  - sending that generated SYS block into the Kin input

### 6. Task artifact archival

- final task-result artifacts now carry metadata such as:
  - `taskTitle`
  - `kinName`
  - `completedAt`
- once a `task_result` artifact is stored in the library, the matching progress runtime is archived out of the active progress panel
- library cards now show clearer task/result provenance

### 7. YouTube transcript queueing

- `<<SYS_YOUTUBE_TRANSCRIPT_REQUEST>>` now supports:
  - `URL` for a single request
  - `URLS` for up to 3 URLs in one sequential batch
- GPT now processes those requests in order
- after each transcript part or queued delivery step, Kin must reply:

```txt
<<SYS_KIN_RESPONSE>>
Received. Send the next part.
<<END_SYS_KIN_RESPONSE>>
```

- only after that ACK does GPT continue to the next part or next queued transcript

### 8. Protocol docs updated

- `lib/taskCompiler.ts`
- `lib/app/kinProtocolDefaults.ts`

Both now explain:

- up to 3 YouTube URLs may be requested in one batch
- Kin should ACK each received transcript step with `SYS_KIN_RESPONSE`
- `TASK_CONFIRM` may use `STATUS: SUSPENDED` for hold state
