# Handoff 2026-05-28

## Scope

This closeout covers the compact transfer controls and GPT chat-tab reply
translation action added after the 2026-05-27 Kin/GPT settings UI cleanup.

## Kin / GPT Transfer Buttons

- The Kin panel `GPTに送る` toolbar action is now a compact icon button.
- The GPT chat-tab `Kinに送る` toolbar action is now the same compact icon
  button.
- The icon uses a one-way curved forward arrow so it no longer collides
  visually with the mobile panel-switch `⇄` control.
- The original labels remain available through `aria-label` and `title`.

## GPT Reply Translation Button

- The GPT chat tab now has a split reply-translation button between
  `返信案` and the compact `Kinに送る` transfer icon.
- The main button displays the selected target language: `EN`, `RU`, or `JP`.
- Clicking the main button sends the current GPT input as a reply draft to be
  translated into the selected language.
- The right-side `▾` control uses a native select overlay so browser dropdown
  behavior works inside the horizontally scrollable toolbar.
- New instruction modes are routed through the normal GPT chat request path:
  `translate_reply_en`, `translate_reply_ru`, and `translate_reply_jp`.
- Server request normalization accepts only those explicit modes and falls back
  to `normal` for unknown values.
- Prompt wrapping for these modes asks for output-only translation, preserving
  meaning, tone, direct-address style, emoji, and line breaks when possible.

## Cleanup Notes

- The toolbar TSX tests were already present but were not included by
  `vitest.config.ts`; the config now includes `GptToolbar.test.tsx` and
  `KinToolbar.test.tsx` explicitly.
- No obsolete instruction mode or hidden alternate route was left behind for
  the reply-translation action.
- Newly added user-facing strings are UTF-8 Japanese/ASCII and passed the
  repository UTF-8 check.
