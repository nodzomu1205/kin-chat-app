# Handoff 2026-05-28

## Scope

This closeout covers the compact transfer controls and GPT chat-tab reply
translation action added after the 2026-05-27 Kin/GPT settings UI cleanup.

## Kin / GPT Transfer Buttons

- The Kin panel `GPTに送る` toolbar action is now a compact icon button.
- The GPT chat-tab `Kinに送る` toolbar action is now the same compact icon
  button.
- The icon uses a one-way curved forward arrow so it no longer collides
  visually with the mobile panel-switch `⇄` control.
- The original labels remain available through `aria-label` and `title`.

## GPT Reply Translation Button

- The GPT chat tab now has a split reply-translation button between
  `返信案` and the compact `Kinに送る` transfer icon.
- The main button displays the selected target language: `EN`, `RU`, `JP`, or
  `IT`.
- Clicking the main button sends the current GPT input as a reply draft to be
  translated into the selected language.
- The right-side `▾` control uses a native select overlay so browser dropdown
  behavior works inside the horizontally scrollable toolbar.
- New instruction modes are routed through the normal GPT chat request path:
  `translate_reply_en`, `translate_reply_ru`, `translate_reply_jp`, and
  `translate_reply_it`.
- Server request normalization accepts only those explicit modes and falls back
  to `normal` for unknown values.
- Prompt wrapping for these modes asks for output-only translation, preserving
  meaning, tone, direct-address style, emoji, and line breaks when possible.

## Explanation Reply-Draft Follow-Up

- Reply-draft follow-ups after `解説` now extract the original source text from
  the latest `[原文]` section before building the reply-draft prompt.
- The extracted original source is used to resolve an explicit reply language
  (`English`, `Russian`, `Japanese`, or `Italian`) when the user's latest
  follow-up does not name a different language. A follow-up such as `Yes` on an
  English `[原文]` therefore sends `Reply language: English.` to OpenAI.
- The previous assistant message's `[日本語訳]`, `[解説]`, and offer text are not
  passed as chat-history context for the follow-up request; the reply target is
  the extracted `[原文]` plus the user's latest instruction.
- If the user explicitly asks for a different language in the follow-up, that
  explicit instruction still applies.
- Reply-draft prompts assume the `[原文]` speaker is a woman and the replying
  user is a man. The same premise is also applied to the direct `返信案`
  (`reply_only`) button.
- This fixes the live regression where answering `はい` to
  `返信案を作りますか？` could bias the reply draft back into Japanese even when
  `[原文]` was English or another language, or could let translation/explanation
  text distort the reply direction.

## Cleanup Notes

- The toolbar TSX tests were already present but were not included by
  `vitest.config.ts`; the config now includes `GptToolbar.test.tsx` and
  `KinToolbar.test.tsx` explicitly.
- No obsolete instruction mode or hidden alternate route was left behind for
  the reply-translation action.
- Newly added user-facing strings are UTF-8 Japanese/ASCII and passed the
  repository UTF-8 check.
- Reply-draft follow-up source extraction now lives in
  `lib/shared/replyDraftFollowup.ts`, keeping client/server follow-up detection
  and source recovery in the same shared boundary.
- The final cleanup rechecked this area for unused helper residue, accidental
  alternate routes, and mojibake. No stale helper names or garbled strings remain
  in the touched chatgpt/reply-draft files.
