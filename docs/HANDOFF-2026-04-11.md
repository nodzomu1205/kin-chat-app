# Kin Chat App 引継書

最終更新: 2026-04-11

## このアプリの位置づけ

このアプリは、Kindroid と ChatGPT を並列に扱うだけのチャット UI ではありません。

目指しているのは、Kin を自然言語フロント、GPT を整理・検索・構造化・記録のバックエンドとして接続し、

- 会話
- タスク化
- 検索
- ライブラリ化
- 文書注入
- Kin 間の受け渡し
- 知識インストール

を一つの連続した作業系として回せるようにすることです。

主役はあくまで Kin 側です。GPT は強い処理系ですが、ユーザー体験の前面には Kin を置く方針です。

## 現在の全体像

### 主要パネル

- Kin パネル
  - Kindroid との会話
  - SYS プロトコルの送受信
  - Kin 切替、接続状態管理
- GPT パネル
  - 通常チャット
  - SerpApi 検索
  - ライブラリ参照
  - メモリ管理
  - タスク支援

### 現在の検索対応

- `google_search`
- `google_ai_mode`
- `google_news`
- `google_local`

`google_maps` は一旦 UI 露出から外しており、場所系は `google_local` で代用しています。

### 現在のプロトコル軸

- `ENGINE`
  - `google_search`
  - `google_ai_mode`
  - `google_news`
  - `google_local`
- `LOCATION`
  - `Japan` のような自然言語表記で指定
  - `google_news` では内部で `gl/hl` に変換
- ライブラリ参照
  - `SYS_LIBRARY_DATA_REQUEST`
  - `SYS_LIBRARY_DATA_RESPONSE`

## 直近で直したもの

### 検索・SerpApi 系

- `ask_ai_mode` の follow-up 継続導線
- `Google Local / Maps` リンクの Google Maps 直接遷移化
- Google Maps 専用カード
- News 専用カード
- Google News の `LOCATION` を query 汚染ではなく locale 解決へ変更
- AI Mode の table 取り込み補強

### トークンと受動参照

- 通常チャットでの受動的な検索履歴注入は停止済み
- 通常チャットの受動参照はライブラリに集約
- `summary_only` ライブラリ注入を軽量化

### メモリ系

- 要約頻度の過敏発火を緩和
- `facts / preferences / collections / topic / goal` の責務を整理し直し
- `worksByEntity` に末尾の雑談文が入る問題を抑制
- 表示名を `named lists` から `collections` 側へ整理

### タスク / ライブラリ

- ライブラリから `タスクに使用` 時、別アイテムが混入する不具合を修正
- `データ取込` でもタイトル未入力のまま仮タイトル生成で開始可能に変更

### プロトコル自動化

設定の `プロトコル` タブに以下の ON/OFF あり:

- A. Kin入力欄の SYS を自動送信
- B. Kin最新レスの SYS を GPT入力欄へ自動転記
- C. GPT入力欄の SYS を自動送信
- D. GPT最新レスの SYS を Kin入力欄へ自動転記

## 現在の注意点

### 1. `sendToGptFlow.ts` はまだ本丸

helper への切り出しは始まっているが、まだ本体が長いです。
次は以下の単位まで分けると保守しやすいです。

- normal chat
- protocol ask / answer
- protocol search
- protocol library
- request assembly
- response post-processing

### 2. `app/page.tsx` もまだ大きい

かなりの副作用は外に出したが、panel wiring と orchestration がまだ集中しています。
次の分割候補:

- task draft orchestration
- panel props composition
- upload / ingest actions

### 3. 文字コード

PowerShell の表示上、日本語が mojibake に見えることがあるが、ファイル自体は UTF-8 の場合がある。
確認時は `Get-Content -Encoding UTF8` を優先。

## プロトコル運用の現在ルール

### 検索

- `SEARCH_REQUEST` では `QUERY` 必須
- `ENGINE` は任意だが、指定するなら上記 4 種に限定
- `LOCATION` は `Japan` など自然表記
- `google_ai_mode` 継続時は query wording を大きく崩さず continuation を活かす

### ライブラリ

- ライブラリ参照は `SYS_LIBRARY_DATA_REQUEST` に一本化
- GPT は `SYS_LIBRARY_DATA_RESPONSE` で index / summary / detail をまとめて返す
- Web で既に取得済みの証拠がライブラリにあるなら、再検索よりライブラリ参照を優先

### タスク送信用 SYS 化

タスク内容から Kin へ渡す SYS フォーマットは、検索時に `ENGINE / LOCATION` を含める前提で整理済みです。
