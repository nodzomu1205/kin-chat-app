## Completed In This Session

### Prefix Commands

The GPT input now supports website-focused commands:

- `Website Map: <url>` / `サイトマップ: <url>`
  - crawls one page
  - displays a readable site map in chat
  - does not save to the library automatically
- `Save Site Map: <url>` / `サイトマップ保存: <url>`
  - saves the current one-page map to the library
- `Get Site Contents: <url>` / `サイト本文取得: <url>`
  - fetches the page and displays extracted main text plus images in chat
- `Download File: <url>` / `ファイル取得: <url>`
  - shows linked files for the page
- `Download and Read File: <url>` / `ファイル読込: <url>`
  - downloads the remote file, runs the existing file-ingest path, and saves
    the extracted document to the library

The map command is intentionally one page at a time. The next page is reached
through the generated `サイトマップ取得` command links in the `Next Links`
table.

### Search And URL Cards

Manual URL cards, normal SerpAPI search cards, and Google News cards now expose:

- `サイトマップ表示`
- `内容表示`

YouTube result cards do not show those website actions. They keep only the
YouTube transcript actions.

### Website Map Output

Website map output was revised for scanability:

- title cells are links, so repeated raw URL columns are avoided
- `Next Links` has per-link `サイトマップ取得` command links
- noisy operational notes such as page/file limit display were removed from the
  ordinary one-page output
- depth labels are no longer shown in the one-page flow
- top and bottom action links provide:
  - `Get Site Contents`
  - `Save Site Map`
  - `Download File`

Saved Website Map library cards render command links through the normal message
content renderer, so the user can continue from a saved map.

### Page Text Extraction

`Get Site Contents` now uses a server-side page-text route backed by the
website-map crawler. The extractor:

- scores likely content containers
- strips common nav/header/footer/aside/script/style boilerplate
- falls back to cleaned full-page text when a selected main container is too
  small
- extracts image links from the selected content scope, falling back to page
  images when needed

This fixed the Mitsui pages where visible body text was initially missed.

### Linked File Flow

Linked files are discovered from the page and can be imported one file at a
time through the existing file ingest pipeline. Website map cards are still only
maps unless the user chooses file import explicitly.

### Cleanup

Final cleanup removed active-code mojibake in:

Remaining mojibake-looking patterns are intentional regression-test patterns.

## Current Boundaries

The Website Map MVP is functional, but it is still a crawler-lite feature:

- it respects same-host crawl scope
- it is one page per command by product choice
- it has timeouts to prevent never-ending loading
- PDF/file bodies are read only after `Download and Read File`
- JavaScript-heavy sites may still need future browser-rendered extraction

Do not reintroduce broad automatic crawling by default. The current UX is
manual, inspectable, and one step at a time.
